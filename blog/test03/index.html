By Puzumei 25 July 2024

Hugo 語法架構

Hugo 是一個基於 Go 語言的靜態網站生成器，非常適合用來建立快速且高效的網站。在這篇文章中， 我們將介紹 Hugo 的基本語法架構，包括配置文件、內容文件、模板文件、短代碼以及靜態資源的使用。

配置文件 (config.toml / config.yaml / config.json)

配置文件用於設置網站的全局配置，例如網站標題、基礎 URL、語言等。以下是一些常見的配置項目：

config.toml

baseURL = "https://example.com/"
title = "My Hugo Site"
languageCode = "en-us"
theme = "your-theme-name"

[params]
  author = "Your Name"
  description = "A brief description of the site."

config.yaml

baseURL: "https://example.com/"
title: "My Hugo Site"
languageCode: "en-us"
theme: "your-theme-name"

params:
  author: "Your Name"
  description: "A brief description of the site."

config.json

{
  "baseURL": "https://example.com/",
  "title": "My Hugo Site",
  "languageCode": "en-us",
  "theme": "your-theme-name",
  "params": {
    "author": "Your Name",
    "description": "A brief description of the site."
  }
}

內容文件 (content/)

內容文件夾中存儲網站的所有內容，每個內容文件通常以 Markdown 文件形式存儲，並包含前端語法（front matter）。

文章範例 (content/posts/my-first-post.md)

---
title: "My First Post"
date: 2024-07-20T12:00:00Z
draft: true
tags: ["tag1", "tag2"]
categories: ["category1"]
---

這是我的第一篇文章內容。

模板文件 (layouts/)

模板文件定義了如何渲染內容，包括頁面模板、部分模板、短代碼等。以下是一些常見的模板文件及其用途：

layouts/_default/baseof.html

<!DOCTYPE html>
<html>
<head>
    <title>{{ .Title }}</title>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
</head>
<body>
    {{ block "main" . }}{{ end }}
</body>
</html>

layouts/_default/single.html

{{ define "main" }}
<article>
    <h1>{{ .Title }}</h1>
    <p>{{ .Content }}</p>
</article>
{{ end }}

layouts/_default/list.html

{{ define "main" }}
<ul>
    {{ range .Pages }}
    <li>
        <a href="{{ .Permalink }}">{{ .Title }}</a>
    </li>
    {{ end }}
</ul>
{{ end }}

layouts/partials/header.html

<header>
    <h1>{{ .Site.Title }}</h1>
    <p>{{ .Site.Params.description }}</p>
</header>

短代碼 (layouts/shortcodes/)

短代碼用於在內容文件中插入自定義元素，使用簡單且靈活。

自定義短代碼範例 (layouts/shortcodes/myshortcode.html)

<p>This is my custom shortcode with parameter: {{ .Get "param" }}</p>

靜態資源 (static/)

靜態資源文件夾中的文件會被原樣複製到網站的根目錄，用於存放圖片、CSS 文件、JavaScript 文件等。

靜態文件範例

static/
└── images/
    └── my-image.jpg

引用靜態資源：

<img src="/images/my-image.jpg" alt="My Image">

進階功能

內容組織（Sections）

Hugo 支持多層次的內容組織，通過文件夾來區分不同的內容部分。

範例

content/
├── posts/
│   ├── post1.md
│   └── post2.md
├── projects/
│   ├── project1.md
│   └── project2.md

多語言支持

Hugo 支持多語言網站，可以在配置文件中設置不同語言的配置。

範例

[languages]
  [languages.en]
    title = "My Hugo Site"
    weight = 1
  [languages.fr]
    title = "Mon Site Hugo"
    weight = 2

部署網站

將 Hugo 生成的靜態文件部署到網頁伺服器上，可以使用 GitHub Pages、Netlify 等平台。生成靜態文件：

hugo

將 public/ 文件夾中的內容上傳到伺服器即可。

總結

Hugo 的語法架構涵蓋了配置文件、內容文件、模板文件、短代碼以及靜態資源的使用。這些組成部分協同工作， 使得 Hugo 能夠快速生成靜態網站，並提供靈活的自定義選項以滿足不同需求。
